我用 AutoCAD 的时候，最先收拾的往往不是界面，而是那些默认开着却基本用不上的联网和启动项。真要提速，很多时候靠的就是把这些细碎设置一项项收干净。下面这几项是我自己最常做的优化，基本都属于低风险、可回退、收益也比较直接的设置。

#### 1. 先确认硬件加速已开启

如果机器本身带独显或核显性能还可以，我会先检查 `硬件加速` 是否已经启用。这个选项对界面缩放、平移、复杂图形刷新会有比较直接的影响。这一步的重点不在于~~开了就一定飞快~~，而在于先把显卡该承担的那部分绘图工作交给显卡。尤其是在较新版本的 AutoCAD 上，如果硬件加速被关掉，很多操作都会显得拖沓。

#### 2. 关闭联网、反馈和信息中心相关功能

AutoCAD 默认会加载不少在线服务、反馈入口和信息中心组件。它们平时未必常用，却会占启动时间，也会让界面多出一些没必要的联网元素。我通常会在管理员控制实用程序里重点关掉下面几类内容：

- 信息中心
- 联机内容
- 联机服务
- 反馈入口

这一步的核心思路很简单：把 CAD 当成本地生产工具来用，而不是一个随时准备联网互动的客户端。只要平时不依赖这些在线功能，关掉以后通常只会更干净、更稳定。

#### 3. 调整启动方式，减少无意义的启动页

这一步对“打开就卡一下”的改善通常很明显，我平时主要会改下面两个命令。

- 修改`StartMode`

在命令行输入：

```text
StartMode
```

把值设为`0`，含义如下：

```text
0 = 禁用开始选项卡
1 = 启用开始选项卡（默认）
```

如果我只是想让软件尽快打开，通常会把它关掉。很多时候启动页本身并不提供真正有价值的信息，反而拖慢进入工作界面的速度。

- 修改`StartUp`

再输入：

```text
StartUp
```

常见取值如下：

```text
0 = 启动和新建时直接进入样板文件
1 = 启动和新建时弹出“创建图形”对话框
2 = 呈现开始选项卡；若开始选项卡被禁用，则空载启动
3 = 预加载功能区并呈现开始选项卡（默认）
```

如果目标就是尽快进入绘图环境，我一般会把 `StartMode=0`，再根据习惯把 `StartUp` 调成更简洁的模式。

> `StartMode` 和 `StartUp` 是配合使用的。前者决定是否显示开始页，后者决定启动后走哪套流程。

#### 4. 从注册表里关闭信息中心和无用导航项

如果前面几步做完后还想再往下压一点启动时间，我才会继续处理注册表。

先按 `Win + R`，输入：

```text
regedit
```

然后展开当前用户对应的 `HKEY_USERS` 项，定位到 AutoCAD 当前版本的配置分支。不同版本路径会略有区别，但思路是一样的。这里通常会处理两类内容：

- 关闭 `InfoCenter`

```text
InfoCenterOn
```
把数值从 `1` 改成 `0`。这样可以关闭标题栏相关的信息中心组件，界面会更干净，启动时也少加载一部分内容。

- 清理 `FileNavExtensions`

如果我从来不用在线协作或 FTP 站点，也会顺手清理 `FileNavExtensions` 下面的这些扩展项：

```text
ACPROJECT
FTPSites
```

这两项有时还会带来界面异常、只剩标题栏、菜单消失之类的老毛病。如果我自己根本用不到它们，通常会直接关掉，省得后面再折腾。

#### 5. 我自己的使用建议

如果不想改太多，我建议优先做下面这几步：

- 开启硬件加速
- 关闭管理员控制实用程序里的联网、反馈、信息中心相关功能
- 把 `StartMode` 设为 `0`
- 根据习惯调整 `StartUp`
- 最后再考虑注册表里的 `InfoCenterOn` 和 `FileNavExtensions`

这几步都是我觉得最值的。比起到处找所谓“加速补丁”，这种自己能看懂、也能随时回退的优化方式会稳很多。
